# Team Agent — 功能需求清单

> 结构化功能清单，作为当前产品范围的唯一真相源。
>
> 优先级说明：`P0 = MVP 必须`，`P1 = MVP 推荐`，`P2 = 后续阶段`

---

## 一、工作区与阶段

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| WS-001 | Workspace | 每个工作区代表一个项目或一次迭代交付上下文 | P0 |
| WS-002 | 双入口 | 支持新项目启动和现有项目迭代两种入口 | P0 |
| WS-003 | 阶段流转 | 工作区按阶段推进，而不是单轮聊天直接结束 | P0 |
| WS-004 | 阶段状态 | 阶段至少支持 draft / awaiting_confirmation / approved / revision_requested / skipped | P0 |
| WS-005 | 阶段确认 | 每个阶段都必须有明确确认动作 | P0 |
| WS-006 | 阶段退回 | 支持对当前阶段发起 revision request | P0 |
| WS-007 | 阶段总览 | 工作区中展示当前阶段、已确认阶段和待处理事项 | P0 |

---

## 二、输入理解与澄清

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| IN-001 | 新需求输入 | 支持从一句自然语言描述创建工作区 | P0 |
| IN-002 | 材料导入 | 支持导入已有需求材料作为迭代输入 | P0 |
| IN-003 | 输入整理 | 系统自动提炼背景、目标、范围线索和约束 | P0 |
| IN-004 | 缺口识别 | 系统指出当前信息缺了什么 | P0 |
| IN-005 | 澄清问答 | 系统围绕关键缺口发起多轮澄清 | P0 |
| IN-006 | 迭代变更识别 | 对已有项目识别本次变更和潜在影响 | P1 |

---

## 三、阶段产物

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| AR-001 | 阶段产物生成 | 每个阶段都要生成正式产物，而不是只有聊天记录 | P0 |
| AR-002 | 产物元数据 | 产物至少记录标题、阶段、版本、状态、生成时间 | P0 |
| AR-003 | 结论摘要 | 每个产物都要包含阶段结论摘要 | P0 |
| AR-004 | 待确认项 | 每个产物都要明确还未确认的内容 | P0 |
| AR-005 | 修订记录 | 每次修订都应被记录 | P0 |
| AR-006 | 交付总览 | 最终提供全部阶段产物的索引列表 | P0 |
| AR-007 | 动态编排 | 产物不使用固定模板，按项目特点动态组织内容 | P0 |
| AR-008 | 稳定骨架 | 虽然不固定模板，但必须有稳定最小结构 | P0 |

---

## 四、阶段定义

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| ST-001 | 需求澄清阶段 | 明确目标、非目标、角色、约束和待澄清问题 | P0 |
| ST-002 | 范围定义阶段 | 明确做什么、不做什么、优先级和范围边界 | P0 |
| ST-003 | 方案设计阶段 | 形成流程、结构、交互和关键规则说明 | P0 |
| ST-004 | 实现准备阶段 | 形成开发说明、拆分建议、风险和验收口径 | P0 |
| ST-005 | 交付总览阶段 | 汇总所有阶段成果、状态和下一步建议 | P0 |

---

## 五、Agent 系统

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| AG-001 | Delivery Lead | 负责推进阶段、收束结论和形成下一步建议 | P0 |
| AG-002 | Analyst | 负责梳理需求、识别缺口、提出澄清问题 | P0 |
| AG-003 | Strategist | 负责范围、优先级和取舍建议 | P1 |
| AG-004 | Reviewer | 负责检查遗漏、歧义、逻辑漏洞和风险 | P0 |
| AG-005 | Spec Writer | 负责把讨论沉淀成正式产物 | P1 |
| AG-006 | Agent 模板 | 支持用户管理交付型 Agent 模板 | P1 |

---

## 六、评审与版本

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| RV-001 | 阶段确认 | 用户可确认当前阶段产物 | P0 |
| RV-002 | 阶段退回 | 用户可要求重写或补充阶段产物 | P0 |
| RV-003 | 版本冻结 | 被确认的阶段产物应形成可追踪版本 | P0 |
| RV-004 | 版本历史 | 支持查看产物的历史版本和修改记录 | P1 |
| RV-005 | 变更对比 | 对迭代项目展示这次变化影响了什么 | P1 |

---

## 七、工作区页面

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| UI-001 | 首页入口 | 支持一句话输入创建工作区 | P0 |
| UI-002 | 工作区列表 | 展示项目、当前阶段和待处理项 | P0 |
| UI-003 | 工作区详情 | 展示阶段流转、对话、产物、确认与修订 | P0 |
| UI-004 | 产物预览 | 在工作区中预览阶段产物 | P0 |
| UI-005 | 产物中心 | 汇总查看全部阶段产物 | P1 |
| UI-006 | 变更视图 | 对迭代项目显示差异、影响和待确认项 | P1 |

## 八、模型与配置

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| CFG-001 | Provider 配置 | 支持配置模型 Provider 和 API Key | P0 |
| CFG-002 | 默认模型 | 支持设置默认模型 | P0 |
| CFG-003 | Fallback | 支持模型失败后的降级链路 | P1 |
| CFG-004 | 用量统计 | 支持查看模型调用量和成本估算 | P1 |

---

## 九、安全与边界

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| SC-001 | 敏感信息保护 | 避免在产物中泄露敏感配置和私密信息 | P0 |
| SC-002 | 高风险操作确认 | 高风险工具调用需要人工确认 | P1 |
| SC-003 | 访问边界 | 限制系统可访问的敏感路径和敏感文件模式 | P1 |

---

## 十、明确不做

| ID | 功能 | 描述 | 优先级 |
|----|------|------|--------|
| NO-001 | 代码生成 | 当前产品不以代码生成作为主能力 | P0 |
| NO-002 | CLI 执行 | 当前产品不提供本地 CLI 执行链路 | P0 |
| NO-003 | 自动改仓库 | 当前产品不直接修改用户代码仓库 | P0 |
| NO-004 | 远程控制 IDE | 当前产品不远程操控本地 IDE 或终端 | P0 |
| NO-005 | 自动测试构建 | 当前产品不承担构建、测试、发布职责 | P0 |
